# 代码经验总结

此技能帮助创建全面的编码经验文档，包括在开发过程中积累的技术见解、最佳实践和问题解决方法。

## 何时调用

- 用户请求记录编码经验或开发见解
- 用户想要创建从编码项目中汲取的经验总结
- 用户询问最佳实践文档
- 用户需要捕获技术知识和解决方案

## 经验文档结构

### 1. 技术经验概览
- 使用的编程语言和框架
- 掌握的关键技术
- 应用的开发方法论
- 使用的工具和环境

### 2. 最佳实践和模式
- 代码组织和结构
- 命名规范和编码标准
- 设计模式实现
- 性能优化技术
- 安全考虑

### 3. 问题解决方法
- 常见错误和调试策略
- 错误处理模式
- 测试方法论
- 代码审查实践
- 重构技术

### 4. 开发工作流
- 版本控制实践
- CI/CD 实现
- 文档编写习惯
- 协作技巧
- 代码维护策略

### 5. 学习与成长
- 关键经验教训
- 克服的挑战
- 发展的技能
- 未来改进领域
- 知识分享方法

## 文档编写指南

### 内容质量
- 提供具体和实际的示例
- 在相关处包含代码片段
- 解释实践背后的"原因"
- 平衡深度和可读性
- 使用清晰、结构化的格式

### 组织结构
- 使用分层标题
- 包含表格进行对比
- 使用项目符号列出清单
- 提供带有解释的代码示例
- 包含资源引用

### 可访问性
- 使用清晰、简洁的语言
- 尽可能避免行话
- 为技术术语提供上下文
- 包含实际示例
- 使其对他人的操作具有指导性

## 模板部分

### 技术技能
- 语言：[列出语言和熟练程度]
- 框架：[列出框架和用例]
- 工具：[列出工具和应用]
- 数据库：[列出数据库技术]

### 代码质量实践
- 遵循的代码风格指南
- 使用的代码检查工具
- 代码审查清单
- 测试覆盖率目标
- 文档标准

### 常见挑战和解决方案
- 挑战 1：[描述] → 解决方案：[方法]
- 挑战 2：[描述] → 解决方案：[方法]
- 挑战 3：[描述] → 解决方案：[方法]

### 性能洞察
- 应用的优化技术
- 跟踪的性能指标
- 瓶颈识别方法
- 可扩展性考虑

### 安全实践
- 身份验证/授权方法
- 数据保护措施
- 漏洞扫描
- 安全编码指南

## 输出格式

生成一个结构良好的 markdown 文档，包括：
- 以执行摘要开始
- 为每个类别包含详细部分
- 提供具体示例和代码片段
- 包含可操作的建议
- 以关键要点和未来方向结束

## 审查清单

在最终确定文档之前：
- [ ] 所有部分完整且连贯
- [ ] 示例准确且相关
- [ ] 代码片段经过测试且可用
- [ ] 最佳实践清晰解释
- [ ] 结构逻辑清晰且易于导航
- [ ] 语言清晰且专业
- [ ] 可操作的见解已突出显示
